## 문제 상황

IndexedDB에 수십~수백 개의 데이터를 한 번에 일괄 저장할 때, 트랜잭션이 롤백되거나 일부 데이터가 유실되는 문제가 있었습니다.

- 여러 put/add를 하나의 트랜잭션에 몰아서 처리
- 중간에 에러(QuotaExceededError 등) 발생 시 전체 트랜잭션이 롤백

## 실제 증상

- 일부 데이터만 저장되고 나머지는 유실
- 저장 중간에 에러 발생 시 전체 트랜잭션이 롤백되어 아무 데이터도 남지 않음

## 원인 분석

- IndexedDB는 트랜잭션 단위로 동작: 하나라도 실패하면 전체 롤백
- 대량 저장 시, 중간에 에러가 발생하면 전체 데이터가 유실

## 실전 대응 패턴

1. **트랜잭션 분할**: 대량 데이터를 여러 개의 작은 트랜잭션으로 나눠서 저장
2. **에러 핸들링**: 각 트랜잭션/데이터별로 성공/실패를 개별적으로 처리
3. **저장 성공/실패 개별 피드백**: 어떤 데이터가 저장됐고, 어떤 데이터가 실패했는지 명확히 표시

### 예시 코드

```js
for (const item of items) {
  try {
    await db.put("store", item);
  } catch (e) {
    // 실패한 데이터만 별도 처리
  }
}
```

## 교훈

- IndexedDB에 대량 데이터를 일괄 저장할 때 트랜잭션 롤백/유실을 반드시 고려해야 한다
- 트랜잭션 분할, 에러 핸들링, 개별 피드백 등으로 데이터 유실을 최소화할 것
- QA/테스트에서 대량 저장 시나리오를 반드시 검증할 것
