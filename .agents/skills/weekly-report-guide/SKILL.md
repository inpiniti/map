# 주간 보고서 작성 실전 스킬

주간 보고서는 팀장·의뢰인·협력부서에게 일주일간의 업무 진행 상황을 명확하게 전달하는 문서입니다. 이 스킬은 누구나 이해할 수 있으면서도 전문성 있게 작성하는 방법을 안내합니다.

---

## 1. 보고서의 기본 구조

주간 보고서는 다음 두 섹션으로 구성합니다:

### [진행 및 완료]
- 현재 진행 중인 업무와 완료된 업무를 정리
- 작업 착수부터 마무리까지 누적된 내용을 기록
- 의뢰사항별로 진행 상황 간략히 표시

### [차주예정]
- 다음 주(또는 다음 기간)에 진행할 업무를 미리 공지
- 현재 진행 중인 업무의 계속 추진 내용
- 신규 예정 업무나 예상 이슈

---

## 2. 항목별 작성 포맷

### 기본 포맷
```
N. 업무 제목 (의뢰번호: XXXXXXXX, ~마감일, 진행상태)
   - 세부 진행내용 1
   - 세부 진행내용 2
```

### 항목 설정 방법

**의뢰번호**: 조직 내부 추적 번호 기재 (없으면 생략 가능)
```
예: 의뢰번호: 2026020140
```

**마감일**: ~YYYY/MM/DD 형식으로 표시 (선택사항)
```
예: ~4/30
```

**진행상태**: 세 가지 중 하나 표시
- **진행**: 현재 진행 중
- **완료**: 주간 내 완료된 업무
- (상태 표기 없음): 완료 또는 완료된 항목

### 세부내용 작성 규칙
- 각 항목당 2~5개의 버릿 포인트로 작성
- 구체적인 액션(무엇을 했는가)을 명확히 기재
- 문제 상황 → 진행 상황 → 결과 순서로 정리
- 너무 상세하지 않되, 알아볼 수 있을 정도로 작성

**Good (간결하고 명확함)**:
```
- 화면 조회 방식 및 구현 방향 검토
- 작업 브랜치 분리 후 개발 착수
```

**Bad (너무 상세함)**:
```
- API 응답 형식을 검토하고 데이터베이스의 join 쿼리와 비교한 후 
  조회 로직을 어디에 둘지 결정했으며 기술팀과 협의해 
  새로운 브랜치 weekly/task123을 생성하고 깃허브에 푸시했다
```

---

## 3. 항목 정렬 순서 (중요!)

### [진행 및 완료] 섹션
1. **우선순위 높음** (긴급, 의존성 많은 작업): 맨 위
2. **우선순위 중간** (진행 중인 작업): 중간
3. **완료된 작업**: 마지막

→ 상황에 따라 **완료된 작업을 맨 위**에 놓으면 성과가 눈에 띕니다.

### [차주예정] 섹션
- 현재 진행 중인 업무의 다음 단계를 먼저 기재
- 신규 업무는 나중에 기재

---

## 4. 작성 팁 & 요령

### 팁 1: 마감일 표기로 일정 관리 강조
```
(~4/30, 진행)  ← 아직 시간이 남았음을 명시
```

이렇게 하면 보고서를 읽는 사람이 일정 압박도를 즉시 파악합니다.

### 팁 2: 진행률이 눈에 띄도록 작성
```
❌ "시설전 등록화면 보완 관련 작업 진행 중"
✅ "시설전 등록화면 보완 관련 작업 (의뢰번호: 2026040037, ~4/30, 진행)
   - 화면 조회 방식 및 구현 방향 검토 완료
   - 작업 브랜치 분리 후 개발 착수"
```

구체적인 액션을 보면 진정한 진행 상황을 알 수 있습니다.

### 팁 3: 문제 상황은 간략하게, 해결 방법은 명확하게
```
❌ 문제 상황만 나열: 이미지 업로드 시 중복 전송 이슈 발생
✅ 문제 + 조치: 이미지 중복 전송 이슈 수정
   (또는 이미지 중복 전송 이슈 원인 파악 및 수정 중)
```

### 팁 4: 미팅·협의는 결과로 정리
```
❌ "관계사와 미팅 실시"
✅ "현장 방문 및 기존 앱 검토"
   또는 "기존 앱 운영 방식 파악 및 개선안 논의"
```

미팅의 결과나 액션이 무엇인지 명시합니다.

### 팁 5: 의존 작업은 상태를 명확히
```
의존성이 있는 작업:
"FMS 개선 (의뢰번호: 2026040037, ~4/30, 진행)
 - A팀 검토 대기 중 → 4/25 승인 예상"
```

### 팁 6: 우선순위 또는 카테고리로 묶기 (선택사항)

숫자나 마크로 일관성 유지:
```
[진행 및 완료]

**진행 중**
1. FMS 시설전 등록화면 보완 ...
2. 가스링크 PDA 편의성 개선 ...

**완료**
1. 가스링크(체납) 에러 처리 ...
```

## 5. 피해야 할 표현

| 피할 표현 | 권장 표현 |
|---------|---------|
| "진행 예정" (다음주 액션 없음) | [차주예정]에만 기재 |
| "검토 중" (정체 상태 의심) | "검토 후 개발 착수" 또는 "검토 예정(4/25)" |
| "이슈 발생" (해결 방법 없음) | "이슈 수정 완료" 또는 "이슈 분석 및 수정 중" |
| "협의 예정" (불확실) | "협의 완료 후 개발 시작" 또는 "관계사와 협의 중" |
| "보조" (책임 불명확) | "지원" 또는 구체적인 역할 명시 |

## 6. 최종 점검 체크리스트

보고서를 작성한 후 다음을 확인하세요:

- [ ] 진행 상황이 누구나 이해할 수 있는가?
- [ ] 마감일이 명확하게 표시되어 있는가?
- [ ] 각 항목의 세부내용이 2~5개 버릿으로 간결한가?
- [ ] 완료된 일과 진행 중인 일이 구분되어 있는가?
- [ ] 다음주 액션이 명확하게 기재되어 있는가?
- [ ] 의뢰번호, 협력팀, 담당자 정보가 필요한 경우 기재되어 있는가?
- [ ] 문제 상황을 간략하게, 조치를 명확하게 기재했는가?

---

## 7. 작성 시간 절약 팁

- **주간 기록**: 요일별로 또는 업무 완료 시마다 간단히 기록해두고, 금요일에 정리
- **템플릿 활용**: 같은 포맷으로 반복되면 템플릿을 만들어 재사용
- **번호 자동화**: 엑셀/구글 시트를 사용하면 번호 매김이 자동화됨
- **의뢰번호 한 곳에 정리**: 협력사 시스템과 연동하면 추적이 쉬워짐

---

> 이 스킬은 의뢰사항을 체계적으로 관리하면서도 읽는 사람이 한눈에 이해할 수 있는 보고서를 작성하는 방법을 안내합니다. 조직 문화에 맞게 조정해서 사용하세요.
